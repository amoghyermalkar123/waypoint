## Waypoint - Query Engine in Rust
This is my first ever serious rust project. It is a Work In Progress query engine written in rust. I have bigger plans for this project but for now it remains a very rudimentary query planning attempt.

## Why
A while back I embarked on a journey to challenge myself as an engineer. After 2-3 years of an on and off relationship with rust, which always ended in me considering myself having skill issues to write even a 10-line program of rust, this time I decided to do approach learning rust differently.

I thought what if I challenged myself to do a much harder thing than learning and building a project in rust? That would make learning rust itself very easy.

So here is my attempt of a query engine project, handwritten, every single character typed, No AI and no LSP used. All of this was me, Zed editor (with LSP settings disabled) and docs.rs.

## Result:
The last few additions to the code is where I started getting comfortable and enabled LSP to avoid typing what i already knew had to be typed. I was baffled by how good rust documentations are.
This journey has been rewarding and I have learned so much about rust and systems engineering and ofcourse query engines.

I maintain a seperate devlog with photos as to how i progressed from an empty unnamed file to this. I plan to start a blog as well!

## Today:
Now that I have internalized quite a few key concepts and a working technique of rust, I have now enabled LSP and do use AI. My usage to AI is limited to redundancy delegation such as adding derive traits to repeating places, updating and helping me maintain tests, etc.

I plan to finish this project as I am following an article series for this.
